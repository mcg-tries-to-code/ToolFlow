## Inspiration credit

ToolFlow was inspired in part by the Hermes work from **Nous Research** and by the broader vision that agents should be able to persist, recover, and improve across longer arcs of work rather than existing only as ephemeral turn-by-turn assistants.

This project does not claim to be Hermes, a fork of Hermes, or a rebranding of Hermes. It is a separate system with its own architecture and emphasis, particularly around:
- bounded workflow execution
- exact-payload approval semantics
- durable ledger-first control-plane state
- recovery and reconciliation
- explicit progress reporting for long-running jobs

The inspiration is real, and so is the divergence.
